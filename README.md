# Road Accident Severity Prediction Using Remote Sensing and Machine Learning

## Overview

This project investigates the impact of environmental and geospatial factors on road accident severity in the United Kingdom. By integrating satellite-derived remote sensing data with official UK road collision records, the study builds machine learning models that predict whether a given accident is likely to result in high severity (fatal or serious injury). The work spans three years of data (2019, 2021, 2022) and demonstrates that incorporating environmental variables meaningfully improves prediction accuracy over conventional traffic-only baselines.

## Problem Statement

Road traffic accidents remain a significant public safety concern. Traditional accident analysis relies primarily on road infrastructure and traffic metadata (speed limit, time of day), neglecting environmental conditions that may influence accident outcomes. This project addresses that gap by fusing geospatial environmental indicators -- Land Surface Temperature (LST), Normalized Difference Water Index (NDWI), and Rainfall -- with accident records to build a more informed severity prediction model.

## Data Sources

- **UK Road Collision Data**: Department for Transport (DfT) road casualty statistics for 2019, 2021, and 2022, containing accident location (latitude, longitude), date, time, collision severity, and speed limit.
- **Remote Sensing Rasters**: Satellite imagery processed through Google Earth Engine to extract per-location environmental variables:
  - **LST (Land Surface Temperature)**: Surface thermal conditions at accident locations.
  - **NDWI (Normalized Difference Water Index)**: Surface moisture and water proximity.
  - **Rainfall**: Cumulative precipitation estimates for the corresponding year.

## Methodology

### Data Preprocessing

1. Loaded and merged three years of DfT collision records.
2. Standardized column names and retained core fields: latitude, longitude, date, time, collision severity, and speed limit.
3. Parsed temporal fields and removed records with missing geospatial or temporal data.

### Environmental Feature Extraction

1. Used `rasterio` to sample raster values (NDWI, LST, Rainfall) at each accident's geographic coordinates for the corresponding year.
2. Merged environmental features with the cleaned accident dataset.

### Feature Engineering

- **Temporal features**: hour, month, day of week, weekend indicator, nighttime indicator.
- **Environmental interaction terms**: wet-heat index (NDWI x LST), rain-wetness index (Rain x NDWI), squared temperature, wetness flag (NDWI > 0).

### Target Variable

Binary classification label (`risk`): 1 if accident severity is fatal or serious (severity <= 2), 0 otherwise.

### Train-Test Split

Temporal split -- 2019 and 2021 data used for training, 2022 data held out for testing. This ensures the model is evaluated on future, unseen data.

### Models

- **Baseline**: Random Forest trained on hour and speed limit only.
- **Random Forest**: 200 estimators, max depth 10, balanced class weights, trained on the full engineered feature set.
- **XGBoost**: 200 estimators, max depth 6, learning rate 0.1, with subsampling and class imbalance handling.
- **LightGBM**: 500 estimators, learning rate 0.05, max depth 8, with early stopping on AUC.

All models are evaluated using ROC-AUC on the 2022 test set.

### Spatial Target Encoding (Advanced)

An additional experiment applies spatial grid-based target encoding by discretizing latitude and longitude into approximately 1.1 km grid cells, computing historical risk rates per grid cell from training data, and using these as an input feature. This captures location-specific accident patterns.

## Requirements

- Python 3.8 or above
- pandas
- scikit-learn
- xgboost
- lightgbm
- rasterio
- geopandas

Install dependencies:

```bash
pip install pandas scikit-learn xgboost lightgbm rasterio geopandas
```

## Usage

1. Place the required dataset files in the project root directory.
2. Open `IAIProject.ipynb` in Jupyter Notebook or JupyterLab.
3. Execute the cells sequentially to reproduce the data cleaning, feature extraction, feature engineering, and model training pipeline.
4. Alternatively, run the standalone scripts:

```bash
python test_lgbm.py
python test_advanced.py
```

## Evaluation Metric

All models are evaluated using **ROC-AUC** (Area Under the Receiver Operating Characteristic Curve), which measures the model's ability to distinguish between high-severity and low-severity accidents across all classification thresholds.
